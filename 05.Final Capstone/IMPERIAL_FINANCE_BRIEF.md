# 🏛️ Imperial Finance – PyPhone Emperor Capstone
**Personal Finance Tracker built entirely on a phone.**

## 🎯 Objective
Build a command‑line application that tracks income, expenses, and budgets.  
Integrate every PyPhone Emperor module into one polished portfolio project.

## 🧱 Modules Integrated
- **Module 01 (Fundamentals):** Variables, input, arithmetic, logical operators.
- **Module 02 (Control Flow):** Loops, conditionals, menus.
- **Module 03 (Strings & Lists):** Data processing, list comprehensions.
- **Module 04 (Functions & Scope):** Modular design, default parameters, `*args`.
- **Module 05 (Dictionaries, Tuples, Sets):** Data storage, category mapping.
- **Module 06 (File Handling & Exceptions):** JSON persistence, error‑safe I/O.
- **Module 07 (OOP):** Classes for Transaction, Budget, Report.
- **Module 08 (Advanced):** Decorators (logging), testing, CSV export.

## 📦 Features
1. **Add Transaction** – amount, category, date, note.
2. **View All Transactions** – filtered by category or date range.
3. **Set Budget** – monthly category limits.
4. **View Budget Status** – progress bars and warnings.
5. **Reports** – monthly summary, category breakdown.
6. **Export to CSV**.
7. **Data persistence** – JSON file (`finance_data.json`).

## 🧪 Technical Requirements
- Use classes: `Transaction`, `Budget`, `FinanceManager`.
- Use JSON for storage (load/save automatically).
- Use custom exceptions for invalid input.
- Use at least one decorator (e.g., `@log_activity` to print action).
- Use list/dict comprehensions where appropriate.
- Provide a menu loop with full error handling.
- Document every function with docstrings.

## 🖥️ Starter Script
The file `imperial_finance.py` contains the skeleton with all classes and menu stubs.  
Your job: implement each method using the skills from the 8 modules.

## 🏆 Success Criteria
- All menu options work without crashing.
- Data persists across sessions.
- Code is readable, commented, and follows PEP8 on a phone screen.
